iOS11和iPhoneX适配

Sep 22, 2017

项目运行打包在真机上,并没有发现需要适配的, 但是使用Xcode9的模拟器运行,结果很是惊讶, 整个界面都不能看. 下面我们来解决这些问题, 我只说我发现的问题

首先定义几个宏
导航栏问题

如果是隐藏系统的导航栏, 为每个控制器添加导航栏的话, 你会发现导航栏的上的子控件整体上移了, 这个是它的父控件上移了, 原来以为是这个是Xcode beta版的bug, 到最后下载正式版的也存在这个问题, 那这个问题就必须要解决了, 找了些资料, 终于找到了解决的办法. 创建一个继承UINavigationBar的子类

这样就解决了每一个控制器的的单独的导航栏的问题

UITableView 问题

偏移

iOS11 控制器的这个属性 automaticallyAdjustsScrollViewInsets就不起作用了 代替它的是 contentInsetAdjustmentBehavior 想知道为什么就Google, 网上可能有用全局设置的

if (iOS11) [[UIScrollView appearance] setContentInsetAdjustmentBehavior:UIScrollViewContentInsetAdjustmentNever];

这样你就会发现一个坑, 如果你的应用里需要访问相册的的话, 你会发现系统的相册会向上偏移, 这个是以为全局设置了这个属性,导致的. 如果没有使用系统的相册的话, 可以这么做. 我们的只需要在基类中设置就行了,

if (iOS11) tableView.contentInsetAdjustmentBehavior = UIScrollViewContentInsetAdjustmentNever;

如果你的项目中没有基类, 那只好一个个控制器中去找了. 这个时候你就会发现基类的重要性. 就当长记性了,未必是一件坏事.

组头和组尾的间距不起作用, 上下拉刷新问题

iOS11 设置这3个属性就行,就会调delegate的返回组头和组尾高度的方法, 不然不会调

导航栏的返回和右边的按钮的点击区域变少了, 还有距离边缘的偏移量多了

返回按钮的点击区域的变少了, 这个可能会带给用户不好的体验, 原来的区域很大, 现在突然变小了. 其实很简单

这样我们就解决了, 这个问题. iOS11 以前我们可以使用

来设置这个返回按钮的距离屏幕的偏移量, 如果你使用的是Xcode9 并且是iOS11 的话, 这样就会发现 返回按钮会偏移很多, 那就这样 navigationItem.leftBarButtonItems = @[backBarItem], 就可以了, 如果你想更加使得返回按钮靠左点的话, 提供一个思路在 layoutSubviews 这里面填加约束.

位置权限

在iOS11，原有的NSLocationAlwaysUsageDeion被降级为NSLocationWhenInUseUsageDeion。因此，在原来项目中使用requestAlwaysAuthorization获取定位权限，而未在plist文件中配置NSLocationAlwaysAndWhenInUseUsageDeion，系统框不会弹出.

都加上吧, 是不会有错的. 就这些了, 希望能带来你一点帮助.
